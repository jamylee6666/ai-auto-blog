## 用 AI 幫你寫 SQL 查詢：非工程師也能查資料庫

## 開頭：為什麼非技術人員需要掌握 SQL 查詢

你是否經常被卡在這個困境裡：想從公司資料庫找出特定資料，但每次都要寄信給IT部門或工程師，等待回覆往往要花上好幾天？或者乾脆放棄，用笨辦法手動搜尋和整理？2026年現在，你根本不需要學三個月的 SQL 語法。用 ChatGPT、Claude 或其他 AI 工具，你可以用自然語言描述需求，AI 直接幫你生成可執行的查詢語句。本文教你如何零基礎上手，省下大量等待時間，甚至讓資料分析工作快上十倍。

## 具體步驟教學

### 第一步：收集資料庫基本資訊（5分鐘）

在要求 AI 寫 SQL 前，先準備這些資訊：

1. **資料庫類型**：是 MySQL、PostgreSQL、SQL Server 還是其他？
2. **相關表名稱**：比如你要查「客戶訂單」，就需要知道存放客戶的表叫什麼、訂單表叫什麼
3. **關鍵欄位名稱**：比如「顧客姓名」這個欄位實際叫 `customer_name` 還是 `cust_name`？

**怎麼快速找到這些資訊？**
- 問公司的 IT 同事或查內部文件
- 登入資料庫工具，點「Schema」或「表結構」查看
- 如果還是不清楚，可以告訴 AI「我只知道我需要找銷售資料」，AI 可以幫你寫探索性查詢先了解資料庫結構

### 第二步：用自然語言向 AI 描述需求（3分鐘）

打開 ChatGPT 或 Claude，使用這個提示範本：

```
我需要查詢公司的 [資料庫類型] 資料庫。
表名稱：[列出相關表]
需求：[用中文清楚描述你想找什麼資料]

例如：我想找出2026年6月份所有超過50萬元的訂單，
並顯示客戶名稱、訂單日期、訂單金額，
結果按金額從高到低排序。

我的資料庫是 MySQL，
表結構是：
- orders 表：order_id, customer_id, order_date, amount
- customers 表：customer_id, customer_name
```

**重點**：越詳細越好。不要只說「幫我查訂單」，而要說清楚：
- 什麼時間範圍
- 什麼條件（金額多少？哪個部門？）
- 要顯示哪些欄位
- 怎麼排序

### 第三步：複製 AI 生成的 SQL 語句（1分鐘）

AI 會給你這樣的結果：

```sql
SELECT 
  c.customer_name,
  o.order_date,
  o.amount
FROM orders o
JOIN customers c ON o.customer_id = c.customer_id
WHERE o.order_date >= '2026-06-01' 
  AND o.order_date <= '2026-06-30'
  AND o.amount > 500000
ORDER BY o.amount DESC;
```

直接複製這段代碼。

### 第四步：貼到資料庫工具並執行（2分鐘）

**用 MySQL Workbench 或 DBeaver 執行查詢：**

1. 登入資料庫
2. 新增查詢頁面
3. 貼入 SQL 語句
4. 按「執行」按鈕（通常是綠色的播放鍵）
5. 幾秒內看到結果

**用企業 BI 工具執行：**
- 如果你公司用 Tableau 或 Power BI，這些工具通常有「SQL 編輯器」或「直接查詢」功能
- 找到「新查詢」選項，貼入 SQL 即可

### 第五步：遇到錯誤時如何修正（3-5分鐘）

常見錯誤：
- **「表不存在」**：檢查表名是否拼錯，或詢問 AI：「我的表可能叫 order_list 或 orders_2026，你能寫個查詢來確認嗎？」
- **「欄位不存在」**：同樣方法，確認欄位名稱
- **「語法錯誤」**：複製完整錯誤信息給 AI，說「執行失敗，錯誤是...，我的資料庫是 MySQL，能修正嗎？」

**修正流程：**
```
我執行你的 SQL 得到這個錯誤：[貼上錯誤訊息]
資料庫類型：MySQL
能幫我修正嗎？也許表名稱或欄位不同。
```

AI 通常能在 10 秒內給你修正版本。

### 第六步：如果結果不對，迭代查詢（2-5分鐘）

假設 AI 給的查詢結果「資料看起來不對」，告訴 AI：

```
查詢結果出來了，但我發現問題：
- 預期應該有500筆資料，結果只有50筆
- 或者某些金額看起來重複計算了

能幫我檢查一下是不是 WHERE 條件寫錯了？
或者需要用 GROUP BY 去重複？
```

AI 會逐步幫你除錯，直到結果符合預期。

### 第七步：儲存查詢供下次使用（1分鐘）

完成後，把 SQL 語句保存到：
- **筆記應用**（OneNote、Notion）：標註用途和執行日期
- **資料庫工具的「已儲存查詢」功能**：大多數工具都有
- **Google Drive 的文字檔**：方便協作時分享給同事

下次遇到類似需求，直接調整時間範圍或條件就行，省掉重新寫查詢的時間。

## 進階用法建議

### 1. **建立「查詢模板庫」加速團隊協作**

把常用的查詢存下來，加上註解，分享給團隊：

```sql
-- 【月度銷售報告】查詢指定月份的銷售總額、客戶數、平均訂單金額
-- 使用方法：改 WHERE 的日期範圍即可
-- 最後更新：2026年6月

SELECT 
  DATE_TRUNC('month', o.order_date) as month,
  COUNT(DISTINCT o.customer_id) as customer_count,
  SUM(o.amount) as total_sales,
  AVG(o.amount) as avg_order_value
FROM orders o
WHERE o.order_date >= '2026-06-01' AND o.order_date <= '2026-06-30'
GROUP BY DATE_TRUNC('month', o.order_date);
```

下次同事需要查詢，直接拿這個改參數，5秒完成，不用每次都重新問 AI。

### 2. **結合 Excel 或 Google Sheets 自動化整理結果**

查詢出來的資料通常是原始格式，把它導出到 Excel 後，可以用 AI（如 Copilot in Excel）進一步：
- 自動製作樞紐分析表
- 生成可視化圖表
- 寫出業務解讀段落

這樣你從「有資料」升級到「有洞察」。

### 3. **進階：寫「複雜查詢」處理多表聯接和計算邏輯**

當查詢涉及3個以上表格、多層條件判斷、或計算字段時，告訴 AI 整個業務邏輯，而不是碎片化描述。例如：

```
我要製作「客戶價值評分報告」，規則如下：
- 查詢過去12個月每個客戶的訂單總額、訂單次數、購買頻率
- 計算得分：總額×40% + 購買次數×30% + 最後購買天數影響×30%
- 只顯示得分 >1000 的客戶
- 按得分排序，附上客戶聯繫方式和對應業務經理

涉及表：customers, orders, order_details, sales_reps
```

AI 會生成一個用 JOIN、WINDOW FUNCTION、CASE WHEN 的完整查詢，這是非工程師手工寫要花半小時的東西，AI 2分鐘搞定。

## 小結

2026年，不會寫 SQL 已經不是問題。用 AI 當你的「資料庫翻譯官」，自然語言進去，可執行查詢出來。這個方法最適合：
- 行銷、銷售想自己拉數據做分析
- 產品經理需要快速驗證某個假設
- 財務或營運常常要生成定期報表

重點是：**表達清楚需求，AI 幫你執行**。剩下的時間和精力，投入在「發現問題」和「決策」，而不是「怎麼查資料」。
