## 用 AI 幫你寫 SQL 查詢：非工程師也能查資料庫

### 開頭：為什麼你需要 AI SQL 查詢工具

你是否曾經面對這個困境：公司資料庫裡有你需要的數據，但要查詢必須寫 SQL 語句，而你根本不懂程式語言？過去你得等工程師有空幫你查，現在浪費了時間，工作效率也跟著降低。事實上，2026 年許多非技術背景的業務、行銷、財務人員都面臨這個瓶頸。好消息是：**AI 已經能夠把自然語言轉換成精準的 SQL 查詢**，讓你像對朋友說話一樣「描述」你想要的資料，AI 就能幫你寫出完整程式碼。

## 步驟教學：從零開始寫出資料庫查詢

### 步驟 1：準備你的資料庫結構資訊

在要求 AI 寫 SQL 前，你需要準備以下資訊：

- **資料庫表格名稱**（例如：users, orders, products）
- **各表格的欄位名稱和型別**（例如：user_id INT, email VARCHAR）
- **表格間的關聯**（例如：users 的 user_id 對應 orders 的 customer_id）

**新手小貼士**：直接問資料庫管理員要「資料庫 Schema 說明文件」，或是用以下查詢語句自己查看：

```sql
SHOW TABLES;
DESCRIBE table_name;
```

把這些資訊複製下來，待會貼給 AI 會更準確。

### 步驟 2：開啟 ChatGPT 並建立新對話

1. 進入 [ChatGPT 官網](https://chat.openai.com)（或使用 Claude、Gemini）
2. 點選「New Chat」開始新對話
3. 在系統訊息中設定角色（重要！這會大幅提升結果品質）

**貼入這個 Prompt**：

```
我需要你協助我撰寫 SQL 查詢語句。
我是一位非程式設計背景的業務人員，需要從公司資料庫中提取數據。

請遵循以下規則：
1. 用繁體中文解釋每個查詢的目的
2. 提供完整、可直接執行的 SQL 語句
3. 如果有多種寫法，說明各自的優缺點
4. 幫我驗證查詢邏輯是否正確
5. 告訴我大概需要多少時間執行（如果資料量很大）

我使用的資料庫系統是：[MySQL / PostgreSQL / SQL Server]
```

### 步驟 3：提供資料庫結構給 AI

在同一個對話中，貼入你的資料庫 Schema：

```
以下是我公司資料庫的表格結構：

【users 表】
- user_id (INT, 主鍵)
- user_name (VARCHAR)
- email (VARCHAR)
- registration_date (DATE)
- user_type (VARCHAR) - 值為 'premium' 或 'free'

【orders 表】
- order_id (INT, 主鍵)
- user_id (INT, 外鍵)
- order_date (DATE)
- total_amount (DECIMAL)
- status (VARCHAR) - 值為 'completed', 'pending', 'cancelled'

【products 表】
- product_id (INT, 主鍵)
- product_name (VARCHAR)
- category (VARCHAR)
- price (DECIMAL)
```

AI 現在已經理解你的資料結構了。

### 步驟 4：用自然語言描述你的需求

這是關鍵一步！**用日常說話的方式描述你想要什麼資料，不用寫程式語言**。

**實際例子**：

❌ **不要這樣問**：
```
SELECT * FROM users WHERE registration_date >= DATE_SUB(CURDATE(), INTERVAL 30 DAY)
```

✅ **應該這樣問**：
```
我想看最近 30 天內註冊的所有用戶名單，以及他們的註冊日期。
並幫我統計有多少人是 premium 會員、多少人是 free 會員。
```

**更多實際例子**：

- 「我需要查出 2026 年 1 月到 3 月之間，總購買金額超過 $5000 的客戶，並列出他們各自買了幾筆訂單」
- 「我想找出電子產品類別中，銷售量最多的前 10 個商品」
- 「最近一個月內，訂單狀態仍是 pending 的客戶有哪些，他們分別等待多少天了」

### 步驟 5：複製 AI 生成的 SQL 並測試

AI 會回傳完整的 SQL 語句。你會得到類似這樣的結果：

```sql
SELECT 
    user_name,
    email,
    registration_date,
    user_type,
    CASE 
        WHEN user_type = 'premium' THEN '付費會員'
        WHEN user_type = 'free' THEN '免費會員'
    END AS 會員類型
FROM users
WHERE registration_date >= DATE_SUB(CURDATE(), INTERVAL 30 DAY)
ORDER BY registration_date DESC;
```

**複製這段語句**到你的資料庫工具（例如 MySQL Workbench、DBeaver、或公司內部的查詢工具），點擊執行。

**新手小貼士**：第一次執行前，在語句最後加上 `LIMIT 5` 限制只顯示 5 筆，確認邏輯沒問題再去掉限制。

### 步驟 6：驗證結果並要求調整

執行後，檢查結果是否符合期望。如果有問題，直接在 ChatGPT 中說明：

```
這個查詢結果不太對。我預期應該有 150 人，但現在只顯示 80 人。
我懷疑是 registration_date 的篩選條件可能有問題。
能幫我檢查並調整嗎？
```

AI 會快速調整查詢邏輯。通常 1-2 次來回就能得到正確結果。

### 步驟 7：保存查詢並建立文件庫

**重要**：把成功的 SQL 語句和對應的說明保存到 Notion、OneNote 或簡單的 Excel 檔案中，建立你的「個人 SQL 查詢庫」：

| 查詢目的 | SQL 語句 | 最後執行日期 | 說明 |
|---------|---------|-----------|------|
| 統計近30天新註冊用戶 | SELECT user_name... | 2026-05-20 | 用於月報表 |
| 找出高消費客戶 | SELECT user_id... | 2026-05-15 | 用於行銷企劃 |

日後遇到類似需求，直接修改幾個日期或條件即可，不用每次都重新寫。

## 進階用法建議

### 建議 1：建立「跨表連結查詢」庫

一旦你掌握基礎後，要求 AI 教你寫涉及多個表格的 JOIN 查詢。例如：

```
我想看每個客戶的名字、他們的總購買次數、平均購買金額，
並按照平均購買金額從高到低排列。
```

這種複雜查詢能解鎖更多分析可能性。

### 建議 2：學會「保存為檢視表 (View)」

請 AI 幫你寫一個檢視表語句，讓複雜查詢變成可重複使用的虛擬表格：

```sql
CREATE VIEW customer_summary AS
SELECT user_id, user_name, COUNT(*) as order_count, AVG(total_amount) as avg_order
FROM orders
GROUP BY user_id;
```

下次直接 `SELECT * FROM customer_summary` 就能快速取得結果。

### 建議 3：與 Excel 或 BI 工具整合

把 AI 生成的 SQL 導入 Power BI、Tableau 或 Google Data Studio，讓查詢結果自動製作成儀表板。結合 AI SQL + 資料視覺化，你就擁有完整的自助分析系統。

## 常見問題排除

**Q: AI 生成的 SQL 會不會有安全風險？**
A: 低風險。建議在測試環境先執行，確認邏輯正確再用於正式資料庫。另外，不要讓 AI 直接連接你的資料庫；應該是人工複製 SQL 到查詢工具執行。

**Q: 如果 AI 寫出的 SQL 執行很慢怎麼辦？**
A: 直接告訴 AI「這個查詢執行了 2 分鐘，資料量有 500 萬筆」，AI 會幫你優化索引建議或改寫查詢邏輯。

**Q: 用哪個 AI 工具最好？**
A: ChatGPT 和 Claude 都可以，建議先試免費版本。Claude 在邏輯推理上略勝，ChatGPT 的對話連貫性更好。選你習慣的即可。
